Overview

What is PRIMO?

PRIMO is a primality proving program based on the ECPP (Elliptic Curve Primality Proving) algorithm. Given positive odd integers, PRIMO tests whether these integers are prime, and if they are it produces primality certificates.
PRIMO is suitable for the checking of crypto-primes and to prove whether they are actually prime... or not.

Latest Release

In order to check whether you have the most recent version of PRIMO, please, visit the PRIMO Homepage.

Getting Started

Text Files

When running, PRIMO works with the following text files:
*.in                            Input file (Integer to test)
primo-xxxxxxxxxxxxxxx-nnn.out   Output file (Primality certificate)
primo-xxxxxxxxxxxxxxx.wr        Work Report file
xx..xx is a unique hexadecimal number that allows to identify all the files created during batch processing.
nnn is an hexadecimal number that indicates the index (when batch processing only) of the tested integer.

Good Practice

In order to certify a number, always create a directory and copy the input file in this directory. Doing so, all the files related to the certification will be located in this directory (while certifying, depending on the size of the number, PRIMO can create hundreds, and even thousands, of temporary files).

First Session

  • Run PRIMO.
  • Open the menu.
  • Click on the New input file... item.
  • Create the new.in file (once created, this file is displayed in a text editor).
  • Write "10^55 + 21" (only the number not the quotes) after "N=".
  • Save the file and close the text editor.
  • Select the Certification tab.
  • Select "2000 dd" with the Number size button.
  • Select "23 bits" with the Divisor size button.
  • Click on the Build prime table button.
  • Click on the Load button.
  • Double-click on new.in.

When the certification is done, a short report is displayed in the editor of the Work Report page. To open a results text file, double-click on the relevant item.

Setup dialog box

CPU Info

CPU Info This group box indicates
  • the name of the processors (in case there are more than one processor on a motherboard, it is assumed they are all identical);

  • the number of physical processors on the motherboard, the number of cores per processor, and the number of hyperthreads per core.

Common

Concurrent tasks... Max number of concurrent tasks.

The value should be chosen according to the number of processors, the number of cores per processor and the number of hyperthreads per core.
For instance, with one processor, four cores per processor and two hyperthreads per core, one could select up to 1*4*2 = 8 tasks.
Hexadecimal strings... Generate hexadecimal strings prefixed with '0x' instead of '$'.

If checked the Pascal hexadecimal tag '$' is replaced with '0x' in certificates. Primo can work (reading/writing) with text files using both tags.

Default value = Unchecked
Reset button This button allows to reset the default value.

Phase 1

Min binary gain
 This parameter allows to increase or to decrease the minimal gain of a test during the pass #1.

This parameter has no effect during the first test of a certification.

Default value = 100%
Elliptic curve tests only
 If checked, the produced primality certificates contain no N+/-1 tests.

Default value = Unchecked
Backtrack
Never before...		 Primo does not try to backtrack before having run the discriminant table whose number is specified.

This directive has no effect during the first test of a certification.

Default value = 2
Backtrack
Always after...		 Primo always tries to backtrack after having run the discriminant table whose number is specified.

This directive has no effect during the first test of a certification.

Default value = 22
Reset button This button allows to reset the default values.

Verification

Obsolete tests... Suppress obsolete tests, if any, when updating certificates.

Due to the new final test of format-4 certificates, when updating certificates from format-3 to format-4, the very last test(s) of a format-3 certificate may be suppressed.

Default value = Checked

Certification Page

The Certification page allows to perform the certification of integers in PRIMO input files and also to resume aborted certificates:
- loading .in files starts certifications ;
- loading .tmp files restarts aborted certifications.

The certificates are always saved to output files in the directory that contains the input files.
While certifying, PRIMO creates and updates temporary files having the .tmp extension. Do not modify them. If ever a .tmp file is modified, all the work done could be lost!

During a certification, the work in progress is automatically saved every 10 minutes.

See also: Primality certificate

Setup

Trial division parameters In order to factor elliptic curve orders, one has to select 2 parameters:
  • Number Size (NS)
    Decimal size of a number to test.

  • Divisor Size (DS)
    Maximal binary size of the prime divisors used by Primo.
When certifying,
  • if the decimal size of a number to test (a candidate or a N value of a certificate test) is greater than or equal to NS, the maximal binary size of the divisors is constant and equal to DS (i.e., Primo makes use of all small primes p such that p < 2DS);

  • while the decimal size of a number to test decreases from NS downto 1000, the maximal binary size of the divisors linearly decreases from DS downto 22;

  • if the decimal size of a number to test is less than 1000, Primo makes use of internal parameters that cannot be modified.
To modify the values, use the arrow keys or click on the buttons and select new values in the menus.

When the NS value is modified, the DS value is automatically set to the default value associated with the new NS value (like indicated in the following table). So, one should always set the NS before setting the DS.

Build prime table Create a prime table containing all the primes having a binary size smaller than or equal to Divisor Size.
It might be necessary to increase the Linux kernel.shmmax value. With Ubuntu, one can do it using the following command line (after having replaced newsize with the value indicated in the PRIMO status line):

sudo sh -c 'echo newsize > /proc/sys/kernel/shmmax'

This modification will last until Linux is rebooted.

Status

Elapsed time Running time of the certification.
Number Number (in the input file list) of the integer that is currently tested.
Phase Number of the current phase (1 or 2).

The Phase 1 consists in finding a list of primes.

The Phase 2 consists in building and factoring class polynomials and in computing elliptic curves and points (which ones prove that the list of primes found during the Phase 1 is indeed a list of primes).
Test Number of the current test.
The value may decrease due to backtracking.
Bits Indicates the binary size of the integer currently tested.
1..48 Miscellaneous information regarding the current status of a certification (each task has its own status line).

Buttons

Load Load files (up to 4095 files) and start certifications (and/or restart aborted certifications). The selected files are always processed in alphabetical order.
Abort Allow to abort a certification.

When a certification is aborted, data that allow to resume it are saved to a file having the .tmp extension. To restart the certification, just load this file.

Verification Page

The Verification page allows to verify and to sign certificates (format-3 and/or format-4). Note that the certificates are always verified and signed while certifying.

When signing, the work in progress is automatically saved every 10 minutes.

See also: Primality certificate

Setup

Check signatures only If checked, PRIMO tries to check the signature only (this allows to quickly reject invalid certificates). If there is no signature section then it verifies the certificate and signs it.

If not checked, PRIMO always verifies the certificate and signs it.

Default value = Checked
Update certificates If checked, PRIMO tries to update an old format-3 certificate to format-4.

If the format-3 certificate is signed and if the Check signatures only option is checked, the creation of the format-4 certificate is very fast.

If the format-3 certificate is not signed or if the Check signatures only option is not checked, PRIMO performs a full verification before creating the format-4 certificate.

Default value = Unchecked

Status

Elapsed time Running time of the verification.
Certificate Number (in the output file list) of the certificate that is currently verified.
Test Number of the current test.
Bits Indicates the binary size of the integer currently checked.
1..48 Miscellaneous information regarding the current status of a verification (each task has its own status line).

Buttons

Load Load files (up to 4095 files) and start verifications. The selected files are always processed in alphabetical order.
Abort Abort a verification.

In order to resume an aborted verification, just reload the .out file.

How to resume a certification?

In the Certification page, click on the Load file(s) and start certification(s) button. Select the .tmp file created when aborting the certification.
That's all.

Input File

A PRIMO input file is a text file that should contain an integer to test. This integer should be greater than 1 and less than 2166016 (i.e. a little more than 50000 decimal digits).

The extension name of an input file must always be .in.

An input file has a syntax examplified by the text within the horizontal bars:
-----------------------------------------
; any comment you want after a semi-colon
[Candidate]
N=
-----------------------------------------
 The file may contain any number of blank lines.
The number (or the expression) that follows N= must be written on a single line.
There must be no character (Space, Tab, or else) before the first character of the section [Candidate], before N= or before a semicolon ;
Expressions are allowed.

Symbol  Meaning  Comment
+ Addition There is no unary +
- Subtraction There is no unary - (but if need be it can be coded as (0-n))
* Multiplication
/ Division Integer division
Example: 8/3 is equal to 2 (not to 2.66666...)
^
**		 Exponentiation The exponentiation is associative on the left: a^b^c = (a^b)^c 
! Factorial Example: 7! is equal to 7*6*5*4*3*2
Spaces before the symbol ! are not allowed
!! Multi-factorial Example: 23!!7 is equal to 23*16*9*2
Spaces before or after the symbols ! are not allowed
# Primorial n# is the product of all the primes less than or equal to n
Spaces before the symbol # are not allowed
$
0x		 Base-16 prefixes Example: $2bc or 0x2bc are equal to 700
Spaces between the prefix and the first digit are not allowed
0..9 Base-10 digits Spaces between the digits of a number are not allowed
0..9
a..f
A..F		 Base-16 digits Spaces between the digits of a number are not allowed
( Opening bracket  
) Closing bracket

In case of error when evaluating an expression, Primo reports either the position of the first invalid symbol or one of the following error codes.

Code   Meaning
-1An exception occurred (user abort, system, ...)
-2Invalid expression
-3The number of closing brackets is not equal to the number of opening ones 
-4Invalid factorial operand (op > 65535)
-5Invalid multi-factorial operands ((op1 < op2) or (op1 ≥ 264) or (op2 < 1))
-6Invalid primorial operand ((op < 2) or (op > 65521))
-7Division by 0
-8Overflow: an intermediate result became greater than or equal to 2531456


Examples of valid input files

;PRIMO Input File

[Candidate]
N=1384435372850622112932804334308326689651568940268408537

;PRIMO Input File
;Number expressed to the base 16

[Candidate]
N=$cc8fa1481b11589c04f066367b204e09132082838aef805f207555505e690153

;PRIMO Input File
;Expression

[Candidate]
N=10^700 + 7

;PRIMO Input File
;Expression with integers expressed to the base 16 (equivalent to 10^700 + 7)

[Candidate]
N=$A^$2BC + $7

Output File (Certificate)

A primality certificate is a sequence of integers that satisfy the requirements of various theorems. With the primality certificate of an integer, an independent verifier can quickly check the primality of this integer (in order to write a verifier, see the verifier-f4.txt file).

Primality certificates are automatically saved to output files (which ones are text files), in the directory that contains the input file.

Example of certificate

[PRIMO - Primality Certificate]
Version=x.x.x - LX64
WebSite=http://www.ellipsa.eu/
Format=4
ID=B346B00DD802B
Created=???-00-0000 00:00:00 AM
TestCount=7
Status=Candidate certified prime

Put here any comment...

2ndPhase=0.01s

Details

[PRIMO - Primality Certificate] Header section
Version= PRIMO version
Website= Homepage of PRIMO
Format= Certificate format
ID= Unique identifier that allows to know all files created during a given work (associated files always have the same identifier)
Created= Creation date of the certificate
TestCount= Total number of tests in the certificate
Status= Status of the certificate

[Candidate] Candidate section 
File= File name of the candidate
Expression= Candidate expressed as a formula (if any in the input file)
N= Candidate (always expressed to the base 16)
HexadecimalSize= Hexadecimal size (base 16) of the candidate
DecimalSize= Decimal size (base 10) of the candidate
BinarySize= Binary size (base 2) of the candidate

[n] Test section 

n is the number of the test
S= number expressed to the base 16
W= number expressed to the base 16
A= number expressed to the base 16
B= number expressed to the base 16
J= number expressed to the base 16
T= number expressed to the base 16
Q= number expressed to the base 16

[Signature] Signature section 
1= 1st part of the signature of a certificate (192 bits)
2= 2nd part of the signature of a certificate (192 bits)

Work Report File

After certifications or verifications, PRIMO opens the Work Report Page and displays miscellaneous information about the work done.
Work reports are automatically saved to .wr files. These text files are always created in the directory that contains the input files (certifications) or the output files (verifications).

Example of work report

[PRIMO - Work Report]
Version=x.x.x - LX64
WebSite=http://www.ellipsa.eu/
Work=Certification
ID=B346B00DD802B
Created=???-00-0000 00:00:00 AM

[Common]
Path=/home/lx64/work/
Selected=1
Processed=1
Certified=1
Candidate 1=Certified, 0.03s

[Candidate 1]
Input=new.in
Output=primo-B346B00DD802B.out
Status=Candidate certified prime

Internal Error

An internal error may be raised by PRIMO for mainly three reasons: a missing/corrupted file, a PRIMO bug or a hardware failure.

In case such an error occurs, try first to redo what you were doing. If the error still occurs, thanks to report the problem with the most possible details: Linux version, processor type, etc. as well as a description of the problem (to get the current e-mail address, see the PRIMO contact page).
